C++几种排序算法

发表于 2019-12-29 | 分类于 public | 阅读次数:

几种排序算法加代码

插入排序Insertion-sort(A)
复杂度：　O(N^2)
author: haining
 data : 2019-10-30

插入排序就像打扑克牌时的抓牌过程一样，假设刚开始手里只有一张牌（数组的开始元素）．

之后我每当摸一张牌（从数组取出下一个元素），就从手里存在的最后一张牌开始，依次往前找,
如果新摸的这张牌比当前看的手里的牌要小，则把当前看的这张牌往后移动一个位置，再继续往前找．　// A[pre+1] = A[pre]

直到看到一张牌X比新摸的这张小，说明新摸的牌应该在这张牌的后边插入．此时X后边的牌都往后移动了一个位置，把新摸的牌直接插入到腾出来的这个位置即可．// A[pre+1] = now;

每次新摸一张牌，都重复上述过程，最终就得到排序后的结果．

这么做的合理性在于，从一开始摸牌就保证了，每次摸牌之前，手里的牌都是已经排好序的，所以可以按次序访问手里的牌，插入到合适的位置．

最差情况下，数组是逆序的，复杂度为 1 + 2 + 3 + 4 + … + N = n(n+1)/2 .一般认为复杂度为 N^2 .

冒泡排序，顾名思义就像是冒泡的过程，遍历ｎ次数组（ｎ为数组元素个数），每次把最小的元素冒泡到正确位置．

过程为：

每次遍历，从数组末尾开始递减进行．也就是说，末尾元素和前一个元素比较，前一元素和前前元素再比较，以此类推．如果后一元素比前一元素小，则两元素位置互换．否则不作操作．
这样，每次比较都会把小数放到前边，最终，就把最小的数推到了最前边．

重复上述过程ｎ次，数组得到排序．注意到这种算法的复杂度是 1 + 2 + 3 + … + n-1 = (n^2)/2 .

快速排序采用分治的思想，任意选定一个数作为锚点（通常是数组的第一个数l，或者最后一个数r，或者中间位置的数(l+r+1)/2）．

这里选定中间位置的数为锚点，遍历数组，那么肯定会存在一个位置，在这个位置之前的数全部小于锚点，位置之后的数全部大于锚点．

这里用了一种比较优雅的找位置的方法，设定了两个指针，同时向中间走，遇到不符合以上要求的位置则停下，两个指针的值交换，然后继续．

找到这个位置之后，把从数组开始到此位置的子数组再进行函数递归调用，这样，直到子数组只剩两个数或者一个数，那么左边的数组就排好了；

再把从该位置到结束位置的子数组进行函数递归调用，同理，右边的数组也就有序了．完成排序过程．

复杂度为　O(Nlog(2/n)).

注意，这种方法会有很多边界问题，不加注意会进入死循环．比如当锚点选择数组第一个数ｌ的时候，递归调用quick_sort(A,l,i－１);quick_sort(A,i,r)就会出错陷入死循环，
可以自己用{1, 2}这个数组跑一下感受一下．这种情况改成(A,l,j);(A,j＋１,r)就不会出问题．同样，当锚点选择数组最后一个数ｒ的时候，
递归调用quick_sort(A,l,ｊ);quick_sort(A,j+1,r)就会出错陷入死循环.

更省事的一种方法可以不用双指针，每次迭代多开两个数组ａ，ｂ，一个存小的数，一个存大的数，然后把a,b作为参数传进递归，一样的效果，只是多了一些开销．
